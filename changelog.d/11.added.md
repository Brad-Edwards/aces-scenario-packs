### Added

- A documentation scrub policy (`docs/scrub-policy.md`) that consolidates the
  scrub-target taxonomy, the remove-or-replace conventions, and the automated
  and review gates that keep canonical docs and migrated content adoption-ready.
- A source-specific scrub checklist in the migration issue template.

### Changed

- The repository scrub sentinel test now reports only offending file paths on
  failure, never the matched downstream term, so private vocabulary cannot leak
  into test output or CI logs.
