# Walkthroughs

**Required-if-present** with the reference triangle (`build/` + `tests/`).
Step-by-step, command-by-command walkthroughs that exercise the same declared
ACES participant behavior as the reference tests — same participant actions,
same expected output, same objective or flag. A walkthrough is live proof and
explanatory guidance, not a second machine-readable scenario specification.

One file per participant route is a good default; name it after the objective or
asset it covers.
