# Walkthroughs

**Required-if-present** with the reference triangle (`build/` + `tests/`).
Step-by-step, command-by-command walkthroughs that **match the test paths** —
same commands, same expected output, same flag. The walkthrough is the
human-readable form of the reference tests.

One file per path is a good default; name them after the path or asset they
cover.
